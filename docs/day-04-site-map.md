# Day 4: Website Page Plan

## Goal

Plan the public church website and the private admin system before building the pages.

## Public Website Pages

### Home

Purpose: Give visitors a warm first impression and help them quickly find service times, location, and the next step.

Main sections:
- Welcome message
- Sunday service timings
- Weekly prayer or fellowship timings
- Upcoming event highlight
- Location and contact shortcut

### About

Purpose: Explain who the church is and what it believes.

Main sections:
- Church story
- Mission and vision
- Beliefs
- Pastor or leadership introduction

### Services

Purpose: Show regular church gatherings clearly.

Main sections:
- Sunday worship
- Bible study
- Prayer meeting
- Youth or children ministry
- Special services

### Events

Purpose: List church announcements and upcoming programs.

Main sections:
- Upcoming events
- Event date, time, and location
- Short description
- Contact person or registration note, if needed

### Sermons

Purpose: Provide a place for sermon videos, audio, or notes later.

Main sections:
- Latest sermon
- Sermon list
- Speaker, date, and topic
- YouTube or audio links

### Contact

Purpose: Help people reach the church easily.

Main sections:
- Church address
- Phone number
- Email address
- Google Maps link
- Contact form

### Donate

Purpose: Explain how people can give safely and clearly.

Main sections:
- Donation instructions
- Giving categories such as tithe, offering, mission, building fund
- Payment details or payment link
- Short note about privacy and records

## Private Admin Pages

### Admin Login

Purpose: Protect the church database so only approved users can access it.

Main features:
- Email and password login
- Protected admin-only pages
- Logout

### Dashboard

Purpose: Give admins a quick overview.

Main features:
- Total families
- Total members
- Donation summary
- Upcoming events

### Families

Purpose: Manage household-level church records.

Main features:
- Add family
- View family list
- Edit family details
- Search families

### Members

Purpose: Manage individual people connected to a family.

Main features:
- Add member
- View member list
- Edit member details
- Track role or membership status

### Donations

Purpose: Record and review giving.

Main features:
- Add donation
- Link donation to a family or member
- Categorize donation type
- Filter by date, donor, and category

### Reports

Purpose: Help church admins understand giving over time.

Main features:
- Monthly totals
- Yearly totals
- Category totals
- Export-ready summaries later

### Events Admin

Purpose: Let admins manage public events.

Main features:
- Add event
- Edit event
- Remove or hide event
- Publish event to public Events page

## First Build Order

1. Home
2. About
3. Services
4. Events
5. Contact
6. Supabase database
7. Admin login
8. Families, members, donations, and reports

## Notes

- Public pages should be simple and mobile-friendly first.
- Admin pages must be private and protected.
- Donation and family data should never be shown publicly.
- Exact church name, service times, address, and contact details can be added when available.
